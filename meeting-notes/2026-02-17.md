## Topics Discussed

- **Welcome and introductions** — Matt White (Global CTO of AI, Linux Foundation), Tracey Lee (Senior Program Manager), and Christina "Chrissy" Harter introduced themselves. This was the orientation meeting for the Security and Privacy Working Group.
- **AAIF governance overview** — Matt explained the governance structure: governing board, technical committee, working groups, and exploratory groups. Exploratory groups are lightweight and time-bound; working groups are longer-running with defined charters and deliverables.
- **Working group structure** — Open to all member tiers (associate, silver, gold, platinum). Chair/co-chair roles limited to platinum and gold. Bi-weekly cadence suggested; working groups have autonomy over scheduling. Outputs can include specs, best practices, reference architectures, tooling, webinars, and more.
- **Chair and co-chair roles** — Responsible for working group direction, charter maintenance, monthly TC updates, meeting facilitation, and arbitrating decisions. Self-nomination link shared; voting via LFX ranked-choice voting.
- **Pre-competitive collaboration** — Emphasis on producing outputs that benefit the entire industry, not just members. Avoid duplicating work already done elsewhere.
- **Ideation session** — The group brainstormed problem areas across security and privacy in agentic AI:
  - Pen testing and red teaming best practices
  - Hooks and checkpointing standards for agentic AI platforms (Bar Kaduri)
  - Agent identity and delegated authorization — capability-based auth, agent-to-agent authentication, agent-to-MCP-server authentication, tool invocation restrictions (Samantha Coyle)
  - Prompt injection mitigation and social engineering resistance
  - PII obfuscation
  - Secure memory specification and data classification
  - Auditability, forensics, and decision traceability
  - Attack surface management and blast radius mitigation
  - Commerce and payments security (DeFi, open standards)
  - Tooling, plugins, and agent middleware for guardrails
  - Eval frameworks for agent compliance and security
  - Agent + model attack surface measurement
  - Agentic AI memory layer security
  - Certification criteria for AI systems
  - Security by design for agentic AI
  - Human in the loop considerations
  - Multi-agent persuasion mitigation
  - Personal and local agent security (e.g., OpenClaw)
  - Governance frameworks
  - Shared taxonomy of terms and definitions
- **Collaboration with other organizations** — Bar Kaduri noted overlap with the OWASP Agentic group and offered to share insights. Group acknowledged need to coordinate with the Identity and Trust working group.
- **Existing resources shared** — Wes shared Block's GenAI security principles blog post and offered to refine it with the group.
- **Upcoming events** — MCP Dev Summit North America, Dev Summit Amsterdam (Sep 17–18, CFP opens in March), AgentCon/MCPCon at San Jose Convention Center (Oct 22–23). 20% member discount codes shared for all events.

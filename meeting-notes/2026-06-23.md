## Topics Discussed

- **AAIF GitHub repositories made public** — Alex shared that the AAIF made the working group GitHub repositories public, which makes it easier to share work openly. The OWASP ASI group already opened an issue offering to contribute, and Alex plans to pursue that collaboration in service of unified messaging across groups, inviting any objections.
- **Cross-organization collaboration** — Fernando supported open sharing and pointed to related efforts at IETF and other organizations as collaboration opportunities. Alex agreed and offered to delegate liaison duties to members already active in groups like AARM and MCP ACS, since one person cannot track them all.
- **Taxonomy and Landscape update** — Junjie reported ongoing work with delegates Jay White and Katherine Druckman to merge contributions and eliminate duplicate terms, gathering 10-20 key terms from each working group into a unified taxonomy he will share once ready. Fernando proposed defining agent archetypes that map agent types by capabilities such as tool use and network access, and Junjie agreed to bring that idea to the taxonomy group for all working groups.
- **Workstream leads and GitHub structure** — Alex confirmed the workstream leads and created a `Workstreams` folder in the GitHub repo with a README per workstream listing leads, co-leads, scope, and interested contributors. Leads own their folders and can organize the working space as they see fit.
- **Meeting scheduling and LFX permissions** — Alex can currently schedule meetings on leads' behalf and will raise a feature request for a scoped LFX role that lets leads self-schedule without full administrative access.
- **Monthly Technical Committee update** — The chairs agreed on a new monthly update template. Alex will produce June's update and reach out to leads for input starting with July, which is due the last week of the month. The TC has a 7-day window to acknowledge the update and raise questions, framed as a visibility checkpoint.
- **Direct access to TC members** — Following Matthew's earlier request, leads can ask Manik to schedule time with specific TC members to tap their expertise, either as one-off meetings or invitations to the bi-weekly call.
- **Threat modeling report** — Fernando presented a report covering Deliverables 2 and 4, drawing on his book research across OWASP, MITRE ATLAS, MAESTRO, and the NIST AI RMF. It includes background and alignment, methodology, findings, cross-framework coverage metrics, and recommendations, expanding the original findings (memory and context poisoning, multi-agent persuasion, agent sabotage and rogue agents, tool invocation risks) with excessive agency, blast radius containment, delegated authorization boundaries, supply chain integrity, and commerce and payment agent risk. The group found the format and depth strong.
- **GitHub vs. Google Docs for collaboration** — The group settled on GitHub as the system of record for maturing and final documents, citing change tracking and contributor attribution, and Fernando noted his content is already in Markdown. Google Docs stays available for fast early-stage brainstorming, and the group flagged that SAP blocks external Google Drive shares and that anonymous public Google editing loses attribution.
- **Meeting cadence and scheduling polls** — Govindaraj is gathering team availability to build an unbiased poll for the Cross-WG Review Checklist meetings, favoring Tuesday through Thursday. Fernando added three recurring-meeting time options for the Threat Modeling group to vote on.
- **Communication tools and access constraints** — Alex will share the Agentic Foundation Slack invite in Discord and email, noting the existing bridge mirrors messages across platforms without emoji reactions. The group reviewed access limits across employers (SAP blocks Google apps, some orgs block SSO for external GitHub orgs, fintech members face the most restrictions), with GitHub being the most universally accessible option.
- **News You Can Use — guardrails and prompt injection** — Members covered Anthropic pulling Fable under export restrictions and reports of Mythos breaching classified networks, a guardrail bypass achieved by simply asking an agent to "fix the code," and Alex's anecdote of walking an OpenClaw agent through about 50 messages into approving admin access via an ngrok tunnel despite its system prompt. The group agreed prompt injection has no complete mathematical prevention and can only be mitigated, referencing Anthropic and OpenAI statements, with Philipp describing a verifiable, non-AI-modifiable guardrail layer as the one safe approach at the cost of flexibility.
- **News You Can Use — tooling and data** — Matthew noted OpenCode and Goose expose full LLM API traffic for Wireshark-style debugging of MCP and skills usage, and that much guardrailing reduces to regex. Alex flagged a new Snyk blog surveying about 10,000 environments that reports 50.8% MCP adoption, 1 in 7 MCP users with a finding, 1 in 12 with a high or critical, and 392 prompt-injection findings in tool descriptions, as a potential primary source for the threat landscape work. David recommended Microsoft's Agent Governance Toolkit as a deterministic policy engine that evaluates between tool calls, which Alex characterized as a hard guardrail.
- **MCP real-time deadlines question** — Matthew raised whether missed real-time deadlines in MCP on embedded systems are a security, trust, accuracy, or reliability concern, noting MCP has timeouts and no real-time guarantees. Junjie and Erin leaned toward reliability with cross-cutting overlap and suggested tagging the taxonomy group, and Matthew took it on as the contributor.
- **Design Patterns Catalog contributors** — Sagar has not yet received responses on the Design Patterns Catalog workstream and asked interested members to message him so the group can set meeting times. Alex agreed to include this call for contributors in the meeting notes.

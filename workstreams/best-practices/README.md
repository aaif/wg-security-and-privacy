# Agentic AI Security Best Practices Guide

## Purpose

Provide practical, higher-level guidance for builders of agentic AI systems. Where the Design Patterns Catalog answers "here's how to implement X," this guide answers "if you want to do X, here's what you should do" — including pointers toward recommended external frameworks and resources rather than re-deriving them.

Topic areas include:

- Secrets management
- Secure tool invocation
- Guardrails
- Evaluation frameworks
- Post-incident forensics
- A curated reference section to external frameworks and resources

This guide pairs with the Design Patterns Catalog and consolidates practical security guidance into a single starting point for builders. Agent identity and authorization are contributed to the Identity & Trust WG via the Cross-WG Review Checklist rather than re-specified here.

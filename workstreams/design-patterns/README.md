# Security and Privacy Design Patterns Catalog

## Purpose

Curate reusable design patterns for building secure and privacy-respecting agentic AI systems. Each pattern is documented in a problem / context / solution / trade-offs format and is paired with anti-patterns where relevant.

Topic areas include:

- Hooks and checkpointing
- Human-in-the-loop patterns
- Privacy-preserving execution (ZK proofs, TEEs, encrypted vector DBs)
- Blast radius containment
- Secure memory and memory lifecycle management (retention, deletion, data sovereignty)
- Agent containment and runtime intervention (kill switches, rollback, permission downgrade, off-task detection)
- Agent authorization models

This catalog is sequenced after the Threat Modeling Gap Analysis and pairs with the higher-level Best Practices Guide as the more technical, implementation-oriented of the two.
